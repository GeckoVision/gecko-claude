## Two ways to execute

### 1. The hosted REST API — `https://api.geckovision.tech`

The contract is [`/openapi.json`](https://api.geckovision.tech/openapi.json); every endpoint is rendered with a "try it" console in the **API Playground** tab, and the guided walkthrough lives in [the verdict API](/api/verdict) + [onboarding](/api/onboarding). The minimal verdict loop:

```bash
# 1. Mint a session for a wallet
curl -s -X POST https://api.geckovision.tech/v1/onboarding/link \
  -H 'Content-Type: application/json' \
  -d '{"wallet_address":"<wallet>"}'        # → { session_token, ... }

# 2. Ask the oracle (the wedge: a pass/defer with surviving dissent + citations)
curl -s -X POST https://api.geckovision.tech/v1/research \
  -H "Authorization: Bearer <session_token>" \
  -H 'Content-Type: application/json' \
  -d '{"idea":"deposit USDC into Kamino","protocol":"kamino","vertical":"dex"}'
```

- **Auth:** `Authorization: Bearer <session_token>` on every call after `link`.
- **Payment:** paid routes settle per-call over **x402** on Solana. In stub mode the path is free; the 402 challenge is advertised at [`/.well-known/x402`](https://api.geckovision.tech/.well-known/x402). See [the verdict API](/api/verdict).

### 2. MCP — add Gecko's tools to your agent

Gecko runs a **hosted MCP server** — no install, just a URL. Add it and your agent gets `gecko_research`, `gecko_trade_research`, and the rest as callable tools.

**Claude Code (one line):**

```bash
claude mcp add --transport http gecko https://api.geckovision.tech/mcp
```

**Any MCP client** (`.mcp.json` / `mcpServers`):

```json
{
  "mcpServers": {
    "gecko": { "url": "https://api.geckovision.tech/mcp" }
  }
}
```

- **Full toolkit + a local agent:** `Read app.geckovision.tech/skill.md` in Claude Code for the skills (trade-coach, trade-agent, grader).
- **Docs MCP:** `https://docs.geckovision.tech/mcp` — search/read these docs as tools.

## The one rule

Gecko is the **verifier**, not the trader. An agent calls it to find out whether
an action is safe and sound *before* acting — and a `pass`/`defer` with surviving
dissent is the product working, not failing. See [Concepts](/concepts).

<Note>Custody is non-custodial throughout — keys live in a TEE or a scoped embedded wallet, and withdrawal is never gated. An agent never hands Gecko a private key.</Note>
